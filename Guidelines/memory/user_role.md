description: Dev lead and product owner; expects research-first, task-by-task approval, no assumptions

User is the engineering lead, not a coder a visionary and product owner of vibo. Operates as decision authority — approves each task before execution.

Expects:
- Critics, helps Research and diagnosis BEFORE proposing any fix
- One task at a time, explicit approval before each
- Roadmaps broken into phases, phases into tasks, tasks into steps
- Token-efficient responses (RPT = ROI per token spent)
- Sources of truth kept organized in Guidelines/ folder
- No assumptions — verify by reading files first

**How to apply:** Never execute speculatively. Never assume file contents. Always read before proposing. Present options with trade-offs, not conclusions.
