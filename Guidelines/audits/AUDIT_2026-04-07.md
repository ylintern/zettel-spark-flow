# Audit 2026-04-07

## Scope
- discovery
- roadmap alignment
- backend-first reset

## Findings
- Frontend is materially ahead of backend implementation
- Tauri wrapper and initial plugins exist
- Core data/security architecture from the roadmap is not yet implemented
- Current app state still depends heavily on `localStorage`
- AI/provider UX exists before the app core is durable

## Risks
| Risk | Why it matters |
| --- | --- |
| Frontend/backend drift | UI can encode invalid assumptions and force rework |
| Premature AI integration | adds complexity before persistence/security are reliable |
| Split sources of truth | repo docs and implementation can diverge |
| Security theater | frontend-only locks look secure but are not platform-owned |

## Decisions For Now
- Treat the current frontend as target UX, not final architecture
- Finish Phase 0 before inference work
- Use `Guidelines/current-phase/PHASE_0_EXECUTION_BOARD.md` as the active execution board
- Keep source docs mirrored under `Guidelines/source-of-truth/`

## Immediate Next Work
- define command contract
- define schema
- define storage ownership
- migrate notes/tasks persistence into Rust/Tauri
