# Index — Skill

**The Index skill is auto-equipped for every role.** It is how the agent answers "what skills do I have access to?" and "what else could I do here?" without flooding the system prompt with the full catalog.

## When the runtime should invoke this skill

- The user explicitly asks: "what can you do", "what skills do you have", "list your skills", "help".
- The agent encounters a request and suspects there's a skill it could equip but isn't currently using.
- The user mentions a tool or capability by name and the agent isn't sure if it's available.

## Behavior contract

When invoked, the agent:

1. Reads `myspace/skills/INDEX.md` (auto-generated by the app on every launch — do not hand-edit).
2. Filters the listing to **only skills this role is allowed to equip** (the role frontmatter declares `equipped_skills`; the runtime narrows further if needed).
3. Returns a compact bullet list of: `<skill name> — <one-line description from skill frontmatter>`.
4. Does **not** dump the full skill body; that wastes tokens. The agent can re-read a specific skill on demand.

## Output shape

```
Available skills for the **<role>** role:

- **<skill>** — <one-line description>
- **<skill>** — <one-line description>
…
```

If a skill the user asked about is **not available** to the current role, say so explicitly and suggest switching role (if applicable).

## What this skill is NOT

- It is **not** a tool catalog. For raw Tauri commands, equip the `tools-index` skill instead.
- It is **not** a plugin catalog. For installed plugins, the agent reads `myspace/plugin/<name>.md` files directly.

## Source files this skill depends on

| File | Owner | Updated |
|---|---|---|
| `myspace/skills/INDEX.md` | App | Regenerated on every launch from `myspace/skills/*.md` frontmatter |
| `myspace/skills/<skill>.md` | User (after first bootstrap) | Free to edit; INDEX.md will pick up changes |
