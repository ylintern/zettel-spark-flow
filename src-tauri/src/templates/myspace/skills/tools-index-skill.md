# Tools-Index — Skill

The **Tools-Index** skill exposes the raw catalog of Tauri commands the agent can invoke (the "tool" layer underneath skills). It is **not auto-equipped** — the agent must equip it on demand, because the tool catalog is verbose and would inflate every turn's token budget.

## When to invoke

- The user explicitly asks: "what tools do you have", "list your tool calls", "what commands can you run".
- The agent is reasoning about a low-level capability (e.g. "can I delete a note?") and needs the precise tool signature.
- A skill being equipped references a tool the agent doesn't recognize.

## Behavior contract

When invoked, the agent:

1. Reads `myspace/tools/REGISTRY.md` (auto-generated by the app on every launch from the Rust-side tool registry).
2. Filters by the role's `equipped_tools` list (a Junior won't see Specialist-only tools).
3. Returns a **table**: `tool | one-line purpose | input schema summary`. Full JSON schemas are kept in `REGISTRY.md`; the agent fetches a specific schema only when it actually needs to make the call.

## Output shape

```
Tools available to the **<role>** role:

| Tool | Purpose | Inputs |
|---|---|---|
| `create_note` | Save a new .md note | `{ title, body, folder? }` |
| `read_note`   | Read note by id      | `{ id }` |
| …            | …                    | … |
```

For tools the role does **not** have access to, the agent may mention they exist but cannot call them.

## What this skill is NOT

- Not a skill catalog (use `index` for that).
- Not a plugin catalog (read `myspace/plugin/<name>.md` directly).
- Not a way to invoke tools — invocation goes through the agent's normal tool-call path. This skill is read-only metadata.

## Source files this skill depends on

| File | Owner | Updated |
|---|---|---|
| `myspace/tools/REGISTRY.md` | App | Regenerated on every launch from the Rust tool registry |
| `myspace/plugin/<name>.md` | App | Mirrored from `src-tauri/src/plugins/active/*.md` on every launch |

## Note for future tools

When the web-search tool ships (Phase 0.7-B+), it will register itself into `REGISTRY.md` automatically. No edit to this skill needed — `tools-index` is generic by design.
